# 无碍e家 网站需求与技术栈

## 需求摘要

| 模块 | 说明 |
|------|------|
| **首页** | 网站名「无碍e家」，陈列三个分类入口 |
| **改造家居** | 肢体/视障/听障三类，每产品：名称、图片、适用场所、适用人群、材料及功能 |
| **政策信息** | PDF 文档，提供下载/预览链接 |
| **经验分享** | 书籍封面 + 简介、作者、读者感悟 |
| **风格** | 蓝白简约 |

## 关于搜索框

**可以加。** 建议在「改造家居」分类页加一个**客户端搜索框**，按名称、适用人群、适用场所等字段筛选，实现简单、无需后端。

## 技术栈方案对比

### 方案 A：Astro（推荐）

静态站点生成器，专为内容/信息陈列类网站设计。

**技术组合：** Astro + Tailwind CSS + JSON 数据 + Pagefind（搜索）

**优点：**
- 默认零 JS 输出，首屏极快
- 表格数据可转为 JSON，直接导入
- 搜索：Pagefind 与 Astro 集成良好，支持中文
- 部署：Vercel/Netlify 一键，免费
- 学习成本适中，文档齐全

**缺点：**
- 需要将 xls/docx 转为 JSON（可写脚本一次处理）

**适合：** 追求高效、快速上线、内容为主的信息站

---

### 方案 B：Next.js 静态导出

若团队已熟悉 React。

**技术组合：** Next.js + Tailwind + JSON + Fuse.js（客户端搜索）

**优点：**
- React 生态，组件丰富
- `next export` 可生成纯静态站点
- 数据同样用 JSON，结构清晰

**缺点：**
- 比 Astro 略重，首屏 JS 稍多
- 构建时间相对更长

**适合：** 团队以 React 为主、未来可能加更多交互时

---

### 方案 C：纯 HTML + Vanilla JS

不依赖框架。

**优点：**
- 无构建步骤，直接打开即可
- 依赖最少

**缺点：**
- 产品/书籍多时，手写 HTML 难维护
- 每次数据更新需改 HTML
- 搜索、分类逻辑要自己实现

**适合：** 产品数很少（例如 <20）、且几乎不更新的情况

---

## 推荐结论

| 优先级 | 方案 | 理由 |
|--------|------|------|
| **首选** | Astro | 信息陈列 + 蓝白简约，正好是 Astro 擅长场景；构建快、部署简单、搜索易集成 |
| 备选 | Next.js | 若更熟悉 React，可接受略多一点复杂度 |
| 不推荐 | 纯 HTML | 数据来自表格，后续更新会很不方便 |

---

## 关键决策

- **架构：** 纯静态站点，无后端、无数据库，全部客户端渲染
- **数据流：** xls → JSON（改造家居产品）、docx/手动整理 → JSON（书籍）、PDF 放 `public/` 目录
- **搜索：** 改造家居页用 Pagefind（Astro）或 Fuse.js（Next.js）做客户端搜索
- **分页：** 改造家居、经验分享列表需分页，每页数量可配置（如 12/24 条）
- **部署：** Vercel 或 Netlify，静态托管，免费额度足够
- **样式：** Tailwind CSS，蓝白主题用配置即可
- **内容更新：** 暂不更新，数据一次性导入即可，无需自动化同步
